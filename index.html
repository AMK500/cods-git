أوامر Git الأساسية والمتقدمة

1. إعداد Git (مرة واحدة عند البدء)

إعداد معلومات المستخدم مثل الاسم والبريد الإلكتروني لضمان أن كل التغييرات يتم تسجيلها باسمك.

git config --global user.name "اسمك"

تعيين اسم المستخدم الخاص بك.

git config --global user.email "email@example.com"

تعيين البريد الإلكتروني الخاص بك.

git config --list

عرض جميع الإعدادات الحالية لـ Git.

2. إنشاء أو استنساخ مستودع (Repository)

الأوامر لإنشاء مستودع جديد أو نسخ مستودع موجود من GitHub.

git init

إنشاء مستودع Git جديد في المجلد الحالي.

git clone [رابط_المستودع]

استنساخ (نسخ) مستودع موجود من عنوان URL المحدد.

3. إضافة الملفات وتسجيل التعديلات

إضافة التغييرات إلى منطقة التهيئة (staging) ثم حفظها في سجل التعديلات (commit).

git add .

إضافة جميع الملفات الجديدة أو المعدلة إلى منطقة التهيئة.

git commit -m "وصف التعديلات"

تسجيل التغييرات مع رسالة توضيحية.

git commit --amend -m "رسالة جديدة"

تعديل آخر تسجيل تم مع رسالة جديدة.

4. ربط المستودع المحلي بالبعيد (GitHub)

ربط المستودع المحلي بمستودع GitHub لرفع التغييرات أو سحبها.

git remote add origin [رابط_المستودع]

إضافة رابط المستودع البعيد مع اسم "origin".

git remote -v

عرض روابط المستودعات البعيدة.

5. رفع التغييرات إلى GitHub

رفع التعديلات من المستودع المحلي إلى المستودع البعيد.

git push origin master

رفع التغييرات إلى الفرع الرئيسي في GitHub.

git push origin اسم_الفرع

رفع التغييرات إلى فرع معين.

git push --force

إجبار التحديث إذا حدث تعارض.

6. سحب التحديثات من GitHub إلى جهازك

سحب التغييرات من المستودع البعيد ودمجها مع الكود المحلي.

git pull origin master

سحب التحديثات من الفرع الرئيسي ودمجها.

git fetch

جلب التغييرات دون دمجها تلقائيًا.
